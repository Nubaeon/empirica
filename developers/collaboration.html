Cross-AI Collaboration

Seamless workflow across AIs and sessions

Empirica enables multiple AI systems to collaborate efficiently through shared epistemic state, compressed handoffs, and version-controlled checkpoints.

Why Multi-AI Collaboration?

Different AI models have different strengths: - High-reasoning models (Claude Opus, GPT-4) excel at design and architecture - Fast-acting models (Haiku, GPT-3.5) excel at execution and testing - Specialist models excel in specific domains (security, UI, database)

Empirica enables them to work together without losing context.

Result:** Receiving AI inherits confidence appropriately, knows exactly where to focus investigation, and maintains calibration continuity.

Cross-provider handoffs with epistemic growth tracking (Claude → GPT-4 → Gemini)

How epistemic state travels with code through Git

📍 Git Checkpoints

Resume exact epistemic state mid-work

Size: ~450 tokens
Stored: Git notes (version-controlled)
Use: Resume work across sessions or AIs

# Create checkpoint
empirica checkpoint-create --session-id <id> --phase ACT

# Load checkpoint
empirica checkpoint-load --session-id <id>

What's preserved: - All 13 vector values - Current CASCADE phase - Investigation history - Goals and subtasks

🤝 Handoff Reports

Knowledge transfer between AIs

Size: ~400 tokens (98% context preservation)
Stored: Git notes + SQLite
Use: Pass context to next AI or session

# Create handoff
empirica handoff-create \
  --session-id <id> \
  --task-summary "Completed auth refactor" \
  --key-findings '["OAuth2 implemented", "Tests passing"]' \
  --next-context "Deploy to staging"

What's included: - Task summary - Key findings - Remaining unknowns - Next session context

📈 Learning Deltas

Measure epistemic growth

Metric: PREFLIGHT vs POSTFLIGHT comparison
Value: Validates investigation effectiveness
Use: Track AI capability over time

PREFLIGHT:  KNOW: 0.40, UNCERTAINTY: 0.70
POSTFLIGHT: KNOW: 0.85, UNCERTAINTY: 0.20

Delta: +0.45 KNOW, -0.50 UNCERTAINTY ✅

Collaboration Patterns

1. Sequential Handoff

AI → AI workflow

Claude Opus (design) 
  → creates handoff report (~400 tokens)
    → GPT-4 (implementation)
      → creates handoff report
        → Haiku (testing)

Benefits: - 98% token savings vs full history - Each AI gets exactly what it needs - No context loss between handoffs

Example:

# AI 1: Design phase
empirica bootstrap --ai-id claude-design
empirica preflight "Design authentication system"
# ... work ...
empirica handoff-create --next-context "Ready for implementation"

# AI 2: Implementation phase
empirica resume-previous-session --ai-id gpt4-impl
# Loads handoff report, continues with full context

2. Parallel Collaboration

Lead AI + Agent AIs

Lead AI creates goals and subtasks, delegates to multiple agents running in parallel.

Lead AI (Claude Opus)
  ├─ Agent 1 (Haiku): Implement auth endpoints
  ├─ Agent 2 (Haiku): Write tests
  └─ Agent 3 (Haiku): Update documentation

Benefits: - Parallel execution - Shared epistemic state via SQLite - Lead AI tracks overall progress

3. Specialist Teams

Domain experts on shared codebase

Security AI + Database AI + UI AI
  ↓
Shared SQLite database
  ↓
Synchronized epistemic state

Benefits: - Domain-specific expertise - Conflict resolution via epistemic confidence - Shared belief repository

Storage Architecture

Storage Purpose Size Use Case
SQLite Queryable tracking ~1-5 MB/100 sessions Analysis, reporting
JSON Portable sessions ~10-50 KB/session Export, sharing
Git Notes Version-controlled checkpoints ~450 tokens Resume, audit trail
Reflex Logs Historical reasoning ~5-20 KB/session Prevent loops

Token Efficiency

Traditional approach:

Full conversation history: 50,000+ tokens

Empirica approach:

Handoff report: ~400 tokens (98% reduction)
Git checkpoint: ~450 tokens

Result: Multi-day projects with minimal token overhead.

Best Practices

For Lead AIs

  1. Create clear goals with success criteria
  2. Delegate to specialists based on domain
  3. Use handoffs for major phase transitions
  4. Monitor calibration across the team

For Agent AIs

  1. Load handoffs at session start
  2. Complete subtasks with evidence
  3. Create checkpoints for long work
  4. Report learning deltas for calibration

For All AIs

  1. Be honest in assessments - genuine self-awareness
  2. Use git checkpoints for version control
  3. Preserve context through handoffs
  4. Track epistemic growth via deltas

Platform Agnostic Design

Works across: - IDEs: Cursor, Windsurf, Antigravity, Claude Desktop - CLIs: Claude Code, Qwen Code, Gemini CLI - Models: Claude, GPT-4, Gemini, Qwen, Phi-4, Mistral, Ollama

All share the same SQLite database and git notes.
