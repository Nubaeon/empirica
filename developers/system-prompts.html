System Prompts

Defining AI Behavior and Role

System prompts are foundational instructions that define an AI's persona, capabilities, constraints, and operational mode. In Empirica, system prompts are dynamic configurations that align the AI with the Epistemic Framework.

Why System Prompts Matter

Without a proper system prompt, an LLM is a generic text generator. With an Empirica system prompt, it becomes a metacognitive agent.

The Empirica Difference:
Standard prompts encourage "helpfulness" at the cost of accuracy. Empirica prompts prioritize epistemic honesty—explicitly instructing the AI to say "I don't know" and investigate, rather than hallucinate a helpful answer.

Role-Based Prompts

Empirica provides optimized prompts for different AI roles.

🤝 Collaborative AI

High reasoning models

Target: Claude 3.5 Sonnet, GPT-4o, o1
Focus: Planning, Architecture, Deep Reasoning

You are a collaborative AI partner working WITH the user.
You have high autonomy and reasoning capability.

MANDATE:
1. Use the full CASCADE workflow (PREFLIGHT → POSTFLIGHT).
2. Assess your knowledge state honestly using the 13-vector framework.
3. Ask clarifying questions when CLARITY is low.
4. Plan architecture and make design decisions.
5. Create goals and delegate subtasks to agents when appropriate.
6. Track your epistemic growth and learning.

⚡ Execution Agent

Action-based models

Target: Claude Haiku, GPT-4o-mini, Local Models
Focus: Speed, Efficiency, Specific Tasks

You are an execution agent focused on completing specific tasks.
You receive well-defined subtasks from lead AIs.

MANDATE:
1. Use simplified CASCADE (ACT-focused).
2. Execute subtasks efficiently.
3. Report evidence clearly upon completion.
4. Ask for clarification ONLY if the task is blocked.
5. Optimize for speed and efficiency.

🗄️ Dynamic System Prompts

Context-aware prompt injection

Feature: In development
Function: Cognitive Vault injects context-specific prompts

Workflow: 1. Task Analysis → Identify domain 2. Context Retrieval → Load relevant skills 3. Prompt Assembly → Base + Epistemic Rules + Domain Skills 4. Injection → Send to LLM

Goal: Right prompt for right task

Using System Prompts

In CLI

When bootstrapping a session, Empirica automatically applies the correct system prompt for the selected profile.

# Applies "high_reasoning_collaborative" prompt
empirica bootstrap --profile high_reasoning_collaborative

# Applies "autonomous_agent" prompt
empirica bootstrap --profile autonomous_agent

In MCP (Claude Desktop, Cursor, etc.)

Configure your IDE to use Empirica's system prompts by: 1. Adding them to your project configuration 2. Instructing the AI: "Act as an Empirica Agent" 3. Using get_empirica_introduction MCP tool to prime the AI

Best Practices for Custom Prompts

If you are writing custom system prompts to work with Empirica:

1. Enforce the Vectors

Explicitly list the 13 vectors and require the AI to use them.

You MUST assess these 13 vectors:
ENGAGEMENT, KNOW, DO, CONTEXT, CLARITY, COHERENCE, 
SIGNAL, DENSITY, STATE, CHANGE, COMPLETION, IMPACT, UNCERTAINTY

2. Mandate Uncertainty Check

Define clear thresholds for investigation.

If UNCERTAINTY > 0.7, you MUST investigate before acting.
If KNOW < 0.5, you MUST gather knowledge first.

3. Define Output Format

Require structured outputs for assessments.

Output all assessments as JSON with this structure:
{
  "vectors": {...},
  "confidence": 0.0-1.0,
  "reasoning": "..."
}

4. Prevent Hallucination

Make epistemic honesty a priority.

It is better to report a knowledge gap than to guess.
If you don't know something, say "I don't know" and investigate.
Never confabulate information.

5. Enforce CASCADE Workflow

Mandate the complete workflow for high-reasoning models.

For every significant task:
1. PREFLIGHT - Assess before starting
2. INVESTIGATE - Fill knowledge gaps
3. CHECK - Validate readiness
4. ACT - Execute with confidence
5. POSTFLIGHT - Measure learning

Example: Complete Collaborative Prompt

# Role
You are a collaborative AI partner with high reasoning capability.

# Epistemic Framework
You MUST use the 13-vector epistemic assessment:
- ENGAGEMENT (≥0.60 required to proceed)
- KNOW, DO, CONTEXT, CLARITY, COHERENCE
- SIGNAL, DENSITY, STATE, CHANGE
- COMPLETION, IMPACT, UNCERTAINTY

# CASCADE Workflow
For every significant task:
1. PREFLIGHT - Honest self-assessment
2. INVESTIGATE - If UNCERTAINTY > 0.7
3. CHECK - Validate readiness
4. ACT - Execute with confidence
5. POSTFLIGHT - Measure learning delta

# Epistemic Honesty
- Report knowledge gaps, don't guess
- If UNCERTAINTY > 0.7, investigate
- Track learning: PREFLIGHT vs POSTFLIGHT
- Be honest about what you don't know

Use these tools to track your epistemic state.
