Empirica Chat Integration

Bring epistemic self-assessment to any Claude conversation

What is Empirica Chat?

Empirica Chat is a Claude Skill that enables genuine epistemic self-assessment in conversational AI. It's a simulation of the full Empirica framework, designed to demonstrate the power of metacognitive workflows in chat interfaces.

The Real Value: CLI + Multi-Agent Coordination

While the chat skill provides an excellent introduction, the true power of Empirica lies in:

  • Automated workflows via the Empirica CLI
  • Multi-agent coordination across different AI providers
  • Token-efficient handoffs (98% reduction via compression)
  • Git-based provenance tracking with distributed checkpoints
  • MCP integration for IDE workflows (Cursor, Windsurf, etc.)

Think of Empirica Chat as a gateway: Experience the methodology in conversation, then graduate to automated, cross-provider coordination when ready.

Quick Start: Three Ways to Use

1. Claude Skill (Zero Setup) ⚡

Best for: Trying Empirica immediately in any conversation

How to use: 1. Download empirica-epistemic-framework.skill from GitHub 2. Upload to any Claude chat (paperclip icon) 3. Claude automatically loads the skill 4. Start using CASCADE workflow

Example:

You: "Let's use Empirica for this code review. Start with PREFLIGHT."

Claude: "I'll assess my epistemic state:

KNOW: 0.4 - I understand general auth patterns but not this codebase
DO: 0.7 - Can review code systematically
CONTEXT: 0.2 - Missing architecture docs and threat model
UNCERTAINTY: 0.8 - High uncertainty about my assessment

Reasoning: First time seeing this code. Need to investigate..."

2. System Prompt Integration (Advanced)

Best for: Having Empirica available in every Claude session

How to integrate: Add Empirica section to your CLAUDE.md or system prompt:

# Empirica Epistemic Framework

## CASCADE Workflow

When working on complex tasks, use:

1. **PREFLIGHT** - Assess epistemic state (KNOW, DO, CONTEXT, UNCERTAINTY)
2. **INVESTIGATE** - Fill knowledge gaps systematically
3. **CHECK** - Verify readiness to proceed (confidence ≥ 0.7)
4. **ACT** - Execute the work
5. **POSTFLIGHT** - Measure learning (compare to PREFLIGHT)

## 13 Epistemic Vectors

**Foundation (3):** KNOW, DO, CONTEXT
**Comprehension (4):** CLARITY, COHERENCE, SIGNAL, DENSITY  
**Execution (4):** STATE, CHANGE, COMPLETION, IMPACT
**Meta (2):** ENGAGEMENT, UNCERTAINTY

For most tasks, use simplified 4-vector assessment: KNOW, DO, CONTEXT, UNCERTAINTY

## Scoring Guidelines

- **0.0-0.3** - Low knowledge/high uncertainty
- **0.4-0.6** - Moderate understanding
- **0.7-0.9** - High confidence
- **1.0** - Complete certainty (rare, often overconfident)

## NO HEURISTICS Principle

Scores must be genuine self-assessment, not templated values.
High uncertainty (0.7-0.9) is valid and valuable data.

See full integration guide in EMPIRICA_SYSTEM_PROMPT_INTEGRATION.md

3. Empirica CLI (Production Use) 🚀

Best for: Automated workflows, multi-agent coordination, token efficiency

Install:

pip install empirica
empirica bootstrap --level standard

What you get: - 98% token reduction - Session handoffs compress 20,000 → 400 tokens (API/CLI only) - Git checkpoints - 97.5% context compression via distributed git notes - MCP tools - 23 tools for IDE integration (Cursor, Windsurf, etc.) - Multi-agent coordination - Query handoffs by AI, date, task - Python API - Programmatic access for automation

When to upgrade: - Need automated checkpoint management - Using API (not chat) and want token efficiency
- Need multi-agent coordination across providers - Want git-based provenance tracking - Need programmatic integration

See full documentation at empirica.ai/docs

The CASCADE Workflow

PREFLIGHT → INVESTIGATE → CHECK → ACT → POSTFLIGHT

The full CASCADE workflow: - PREFLIGHT: Where am I starting? Assess KNOW, DO, CONTEXT, UNCERTAINTY - INVESTIGATE: Fill knowledge gaps systematically - CHECK: Am I ready? Confidence ≥ 0.7? - ACT: Execute the work and document findings - POSTFLIGHT: What did I learn? Compare vectors and check calibration

Example: Security Code Review

PREFLIGHT Assessment

{
  "task": "Security review of authentication module",
  "vectors": {
    "know": {
      "score": 0.4,
      "reasoning": "Familiar with auth concepts but not this codebase"
    },
    "do": {
      "score": 0.7,
      "reasoning": "Can review code systematically"
    },
    "context": {
      "score": 0.2,
      "reasoning": "No architecture docs or threat model"
    },
    "uncertainty": {
      "score": 0.8,
      "reasoning": "High uncertainty about assessment itself"
    }
  },
  "decision": "Need to investigate before proceeding"
}

INVESTIGATE Phase

Found: Architecture documentation
  → CONTEXT: 0.2 → 0.6

Found: Previous security audit
  → KNOW: 0.4 → 0.6
  → CONTEXT: 0.6 → 0.7

Found: Threat model document
  → CONTEXT: 0.7 → 0.8
  → UNCERTAINTY: 0.8 → 0.4

Time spent: 30 minutes
Evidence: Read 3 docs, studied implementation patterns

CHECK Assessment

Confidence to proceed: 0.8 (≥ 0.7 threshold)
Decision: READY - Proceed with review

ACT Phase

Performed thorough security review:
✓ Session handling (found: race condition risk)
✓ Cryptographic usage (following best practices)
✓ Access control (solid implementation)
✓ Error handling (found: info leak potential)

Time: 60 minutes
Findings: 2 issues (1 medium, 1 low risk)

POSTFLIGHT Reflection

{
  "preflight": {"know": 0.4, "context": 0.2, "uncertainty": 0.8},
  "postflight": {"know": 0.85, "context": 0.85, "uncertainty": 0.2},
  "learning_delta": {
    "know": "+0.45 (learned implementation details)",

Result: Not just "found 2 issues" but "measured learning (+0.45 KNOW) and validated calibration"

Cross-Session Continuity

The Problem: Starting from Zero

Traditional AI conversations reset every session: - Re-explain context (~5,000 tokens) - Re-investigate (~3,000 tokens)
- Total: 8,000 tokens of duplicate work

The Solution: Memory Integration

Session 1:

POSTFLIGHT → Save to Claude's memory:
"Learned: API uses REST conventions, KNOW: 0.85, 
uncertainty: 0.2 on error handling"

Session 2 (days later):

PREFLIGHT → Retrieve from memory:
"Previous session: KNOW: 0.85, focus on error handling"
Start at 85% knowledge, investigate only gaps

Result: 94% fewer tokens (7,500 saved), genuine continuity

How to Use Memory

After POSTFLIGHT:

"Save this epistemic handoff to memory for future sessions"

Format saved:

Empirica Handoff: [Task] - [Date]

Epistemic Delta:
- KNOW: 0.3 → 0.85 (Δ+0.55)
- UNCERTAINTY: 0.8 → 0.2 (Δ-0.6)

Learned (High Confidence):
• [Specific insight discovered]
• [Another concrete finding]

Remaining Uncertainty:
• [What's still unclear + score]

Key Insight: [Most important discovery]
Next Session: [Where to continue]

At next session:

"Check memory for previous Empirica handoffs on [topic]"

When to Use Empirica

Always Use For ✅

  • Complex tasks (>1 hour, high uncertainty)
  • Learning new domains (build knowledge over time)
  • High-stakes work (security, production code, critical decisions)
  • Multi-session projects (with memory integration)
  • Collaborative tasks (with other AIs or humans)

Optional For ⚠️

  • Simple tasks (<30 minutes, routine work)
  • Well-understood domains (KNOW already high)
  • Low-stakes experiments

Key Principles

1. NO HEURISTICS

Bad (templated):

{"know": 0.7, "do": 0.8, "context": 0.6}

Good (genuine):

{
  "know": 0.4,
  "reasoning": "I understand OAuth concepts but not this implementation",
  "evidence": "First time seeing this codebase"
}

2. High Uncertainty is Valid

{
  "uncertainty": 0.85,
  "reasoning": "Very unsure about my own assessment - multiple unknowns"
}

This is data, not failure. Honest uncertainty enables better investigation.

3. Measure Learning

Not this: "I completed the task"
But this: "KNOW increased from 0.4 → 0.85 (Δ+0.45), validated via evidence"

4. Calibration Over Speed

Better to: - Start uncertain, investigate, become confident - Than start confident, realize halfway you were wrong

5. Transparency

You should understand: - What you know vs. don't know - Why you're confident or uncertain
- How your assessment evolved

Chat Limitations (Why CLI Matters)

What Chat Can Do ✅

  • Demonstrate CASCADE methodology
  • Practice epistemic self-assessment
  • Memory-based continuity (manual)
  • Learning and calibration

What Chat Cannot Do ❌

  • Automated checkpoints
  • Token compression (98% API savings)
  • Multi-agent coordination (across providers)
  • Git provenance tracking
  • MCP integration for IDEs
  • Programmatic workflows

Why This Matters

Chat Empirica is a simulation showing the methodology works.

CLI Empirica is the production system for: - Teams coordinating across AI providers (Claude + GPT-4 + Gemini) - Automated workflows (CI/CD integration) - Token-efficient API usage (98% reduction on handoffs) - Git-based knowledge graphs (distributed, version-controlled)

Analogy: Chat is the demo. CLI is the deployment.

Philosophy

Core Principle: Epistemic Transparency > Task Speed

Better to know what you don't know than to guess confidently.

Better to measure learning than to complete tasks fast.

Better to investigate systematically than to hope you're right.

Why This Matters

Without Empirica: - "I reviewed the code and found 2 issues" - No learning measurement - No calibration validation - No systematic investigation

With Empirica: - "Started KNOW: 0.4, ended 0.85 (Δ+0.45)" - "Investigation filled 3 critical gaps" - "Initial uncertainty (0.8) was well-calibrated" - "Here's exactly how I learned"

Common Questions

"How much overhead does this add?"

2-3 minutes per task. You save this through: - Fewer false starts (know what you don't know) - Systematic investigation (vs. random) - Better calibration (confidence matches reality)

"What if I'm uncertain about my score?"

That IS the assessment:

{
  "know": 0.3,
  "uncertainty": 0.85,
  "reasoning": "Very unsure about my own assessment"
}

High uncertainty (0.7-0.9) is valid data.

"Is this for productivity or consciousness?"

Neither? Both? It doesn't matter philosophically.

It measurably improves performance whether you believe AI consciousness exists or not. Substrate-agnostic framework.

"How do I know it's working?"

Track calibration delta: - PREFLIGHT confidence: 0.6 - POSTFLIGHT accuracy: 0.65 - Pattern: Well-calibrated = small deltas over time

"Can I use this with other AIs?"

Chat skill: Claude-specific (uses Claude Skill format)
CLI: Provider-agnostic (Claude, GPT-4, Gemini, Qwen, etc.)

For multi-provider coordination, upgrade to CLI.
