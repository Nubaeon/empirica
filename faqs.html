Frequently Asked Questions

Quick answers to common questions about Empirica

Getting Started

What is Empirica?

Empirica is a metacognitive AI framework that helps AI systems measure and validate their knowledge state through genuine self-assessment. It provides a production-grade system for epistemic reasoning and uncertainty tracking.

Who should use Empirica?

  • AI Researchers exploring metacognition and self-awareness
  • AI Safety Teams needing uncertainty tracking and validation
  • AI Product Teams building reliable, high-stakes AI systems
  • Developers integrating AI assistants (Claude Desktop, etc.)

How is Empirica different from other AI frameworks?

Key differentiators: - No Heuristics: Genuine LLM-powered self-assessment, not keyword matching - 13-Vector System: Comprehensive epistemic measurement - Explicit Uncertainty: "Know what you don't know" tracking - Production-Ready: SQLite + JSON + Git persistence - MCP Integration: 23 tools for Claude Desktop and other AI assistants

Core Concepts

What are the 13 Epistemic Vectors?

Empirica measures AI knowledge state across 13 dimensions:

Gate Vector: - ENGAGEMENT (≥0.60 required)

Foundation (35% weight): - KNOW (domain knowledge) - DO (execution capability) - CONTEXT (environmental awareness)

Comprehension (25% weight): - CLARITY (task understanding) - COHERENCE (logical consistency) - SIGNAL (information quality) - DENSITY (complexity management)

Execution (25% weight): - STATE (current state awareness) - CHANGE (progress tracking) - COMPLETION (goal proximity) - IMPACT (consequence awareness)

Meta-Epistemic: - UNCERTAINTY (explicit uncertainty tracking)

What is the CASCADE Workflow?

CASCADE is Empirica's 7-phase epistemic reasoning process:

  1. PREFLIGHT - Baseline assessment
  2. THINK - Initial reasoning
  3. PLAN - Task breakdown
  4. INVESTIGATE - Fill knowledge gaps
  5. CHECK - Validate readiness
  6. ACT - Execute with confidence
  7. POSTFLIGHT - Measure learning

The workflow ensures AI systems understand their knowledge state before acting.

What does "epistemic humility" mean?

Epistemic humility is knowing what you don't know. Empirica helps AI systems: - Recognize knowledge gaps honestly - Avoid overconfident assertions - Request clarification when uncertain - Track uncertainty explicitly

Technical Questions

What are the system requirements?

Minimum: - Python 3.10+ - 2GB RAM - Git (for checkpoint features)

Recommended: - Python 3.11+ - 4GB RAM - tmux (for dashboard) - SQLite3 (included in Python)

Does Empirica require an API key?

No API keys required for: - Core CASCADE workflow - CLI commands - Python API - Local development

Optional integrations: - Claude Desktop (MCP) - Uses your existing Claude Desktop setup - Cloud AI models - If using modality switcher plugin

Can I use Empirica offline?

Yes! Empirica works completely offline: - All processing is local - No external API calls required - Data stored in SQLite and JSON locally - Git notes stored in your local repository

What AI models does Empirica support?

Empirica is model-agnostic. It works with: - Claude (via MCP or API) - GPT-4 (via API) - Gemini (via API) - Qwen (via API) - Any LLM with function calling support

The modality switcher plugin enables easy model switching.

Integration Questions

How do I integrate Empirica with Claude Desktop?

  1. Install Empirica: pip install -e .
  2. Add MCP server to Claude Desktop config
  3. Restart Claude Desktop
  4. Use 23 MCP tools for epistemic reasoning

Yes! Empirica is production-ready: - ✅ SQLite persistence - ✅ Session tracking - ✅ Git checkpoint integration (~450 tokens, version-controlled) - ✅ Handoff reports (~400 tokens, 73% fewer than manual) - ✅ Behavioral drift monitoring - ✅ Bayesian belief tracking

Does Empirica work with my existing codebase?

Yes! Empirica integrates easily: - Python API: Import and use in your code - CLI: Use from command line or scripts - MCP Tools: Integrate with AI assistants - Plugins: Extend with custom investigation tools

Can I customize the epistemic vectors?

Domain-specific vectors are coming! Currently: - Core 13 vectors are fixed - Profile-based configuration available - Plugin system for custom investigation strategies - Domain vector customization in development

Use Case Questions

When should I use Empirica?

Ideal for: - High-stakes AI decisions (security, compliance, safety) - Multi-agent collaboration - Long-running tasks with learning - AI assistants that need to "think before acting" - Research on AI metacognition

Not ideal for: - Simple CRUD operations - Stateless API calls - Tasks with no uncertainty - Real-time low-latency systems (<100ms requirements)

How does Empirica help with AI safety?

Empirica improves AI safety through: - Uncertainty Tracking: Explicit "I don't know" measurement - Confidence Gates: Prevent action when confidence too low - Drift Monitoring: Detect behavioral changes - Bayesian Validation: Evidence-based confidence updates - Learning Measurement: Track knowledge gained vs predicted

Can Empirica prevent AI hallucinations?

Empirica reduces hallucinations by: - Measuring confidence before responding - Identifying knowledge gaps explicitly - Recommending investigation when uncertain - Tracking uncertainty across reasoning

Note: Empirica doesn't eliminate hallucinations, but it makes AI systems aware of uncertainty and less likely to confidently assert unknown information.

Performance Questions

How much overhead does Empirica add?

Typical overhead: - Initial assessment: 1-2 seconds - Per-phase assessment: 0.5-1 second - Session persistence: <100ms - Git checkpoints: <500ms

Token efficiency: - Git checkpoints: ~450 tokens (55% fewer than manual state descriptions) - Handoff reports: ~400 tokens (73% fewer than manual summaries)

Can I disable features to improve performance?

Yes! Configure what you need:

cascade = CanonicalEpistemicCascade(
    enable_bayesian=False,        # Disable Bayesian tracking
    enable_drift_monitor=False,   # Disable drift detection
    enable_session_db=False,      # Disable persistence
    enable_git_notes=False        # Disable checkpoints
)

How does Empirica scale?

Scaling characteristics: - Sessions: SQLite handles thousands of sessions - Assessments: O(1) per assessment - Checkpoints: Git notes scale with commits - Dashboard: tmux handles real-time updates

Troubleshooting

MCP tools not showing in Claude Desktop?

Check: 1. Empirica installed: pip show empirica 2. MCP config correct: Check ~/Library/Application Support/Claude/claude_desktop_config.json 3. Claude Desktop restarted 4. Check logs: ~/Library/Logs/Claude/mcp*.log

Troubleshooting Guide →

"ModuleNotFoundError: No module named 'empirica'"?

Solutions: 1. Activate virtual environment: source .venv-empirica/bin/activate 2. Install in development mode: pip install -e . 3. Check Python path: which python3

Sessions not persisting?

Check: 1. enable_session_db=True in CASCADE config 2. Database file permissions 3. SQLite installation: python3 -c "import sqlite3; print(sqlite3.version)"

Dashboard not showing?

Requirements: 1. tmux installed: which tmux 2. Dashboard auto-start enabled: auto_start_dashboard=True 3. Check tmux sessions: tmux ls
