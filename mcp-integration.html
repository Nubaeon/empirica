MCP Server Integration

Enhance Any AI Environment with Epistemic Intelligence

The Model Context Protocol (MCP) Server is the bridge between Empirica and any MCP-compatible AI environment. It gives your AI assistant genuine epistemic self-awareness through 23 specialized tools.

Overview

What is MCP Integration?

MCP (Model Context Protocol) is a standard that allows AI assistants to access external tools and data sources. Empirica's MCP server exposes the entire CASCADE framework as tools that any AI can use to assess its own knowledge state.

Tested across:

Modern IDEs: Cursor, Windsurf, Antigravity, Claude Desktop
CLI Tools: Claude Code, Qwen Code, Gemini CLI, Copilot CLI, Grok CLI
AI Models: High-reasoning (Claude, GPT-4, Gemini) and fast-acting (Qwen-3-32B, Phi-4, Mistral, Ollama)

Key Benefits

  • Seamless Integration: Works with any MCP-compatible environment
  • Context Awareness: The AI knows your project's epistemic state
  • Tool Access: Gives the AI 23+ specialized tools for assessment and investigation
  • Session Persistence: Maintains context across chat sessions
  • No Code Changes: Drop-in enhancement for existing AI workflows

Installation & Setup

1. Install Empirica

First, ensure you have Empirica installed:

# From source
git clone https://github.com/Nubaeon/empirica.git
cd empirica
pip install -e .

# Or via pip (coming soon)
pip install empirica

2. Configure Your MCP Client

Add the following to your MCP client's configuration file:

Claude Desktop (~/Library/Application Support/Claude/claude_desktop_config.json on macOS):

{
  "mcpServers": {
    "empirica": {
      "command": "/path/to/empirica/.venv-mcp/bin/python3",
      "args": ["-m", "empirica.mcp_local.empirica_mcp_server"]
    }
  }
}

Cursor/Windsurf (.cursor/mcp_config.json or .windsurf/mcp_config.json):

{
  "mcpServers": {
    "empirica": {
      "command": "empirica",
      "args": ["mcp", "start"]
    }
  }
}

Antigravity (automatically detects MCP servers in your environment):

# Just ensure empirica is in your PATH
which empirica

3. Restart Your MCP Client

Restart your MCP client (Claude Desktop, Cursor, etc.). You should see a connection confirmation or the tools becoming available.

Verification: Ask your AI assistant: "What Empirica tools do you have access to?"

It should list all 23 MCP tools.

Available MCP Tools (23 Total)

The MCP server exposes a comprehensive suite of tools organized by function:

🛠️ Session Management

Control your epistemic lifecycle.

  • bootstrap_session(ai_id, session_type, profile): Initialize a new metacognitive session

  • resume_previous_session(ai_id, count): Load context from past work

  • get_session_summary(session_id): Retrieve high-level session data

  • get_epistemic_state(session_id): Check current vector values

🔄 Assessment Workflow (CASCADE)

Run the complete CASCADE process.

  • execute_preflight(session_id, prompt): Run initial assessment

  • submit_preflight_assessment(session_id, vectors): Record initial state

  • execute_check(session_id, findings, ...): Validate readiness

  • submit_check_assessment(session_id, vectors, ...): Record check state

  • execute_postflight(session_id, task_summary): Run final reflection

  • submit_postflight_assessment(session_id, vectors): Record learning

🎯 Goals & Subtasks

Track progress and structure work.

  • create_goal(session_id, objective, ...): Define a new objective

  • add_subtask(goal_id, description, ...): Break down goals

  • complete_subtask(task_id, evidence): Mark work as done

  • get_goal_progress(goal_id): Check completion status

  • list_goals(session_id): View all goals

💾 Continuity & Handoffs

Preserve context across sessions.

  • create_git_checkpoint(session_id, ...): Save state to git notes (~450 tokens)

  • load_git_checkpoint(session_id): Restore state from git

  • create_handoff_report(...): Generate summary for next AI (~400 tokens)

  • query_handoff_reports(...): Search past handoffs

📚 Help & Guidance

Get contextual assistance.

  • get_empirica_introduction(): Primer on the framework

  • get_workflow_guidance(phase): Instructions for specific phases

  • cli_help(): CLI command reference

  • get_calibration_report(session_id): View learning metrics

Usage Examples

1. Starting a Session

User: "I need to debug this authentication issue."

AI (using MCP): 1. Calls bootstrap_session(ai_id="claude", session_type="development") 2. Calls execute_preflight(prompt="Debug auth issue") 3. Assesses its own knowledge across 13 vectors 4. Submits submit_preflight_assessment with honest self-evaluation

Result: The AI now knows what it knows, what it doesn't know, and what it needs to investigate.

2. Strategic Investigation

User: "The error seems related to the token refresh logic."

AI (using MCP): 1. Creates a goal: create_goal(objective="Fix token refresh bug") 2. Adds subtasks: - add_subtask(description="Analyze refresh_token function") - add_subtask(description="Check token expiration logic") - add_subtask(description="Verify error handling") 3. Investigates systematically, marking subtasks complete with evidence

Result: Structured investigation with tracked progress and evidence.

3. Handoff Between Sessions

User: "Let's pause here and continue tomorrow."

AI (using MCP): 1. Calls execute_postflight to reflect on what was learned 2. Calls create_handoff_report with: - Task summary - Key findings - Remaining unknowns - Next session context

Result: ~400 token handoff report that preserves 98% of context for the next session.

4. Multi-Session Continuity

Next Day:

User: "Let's continue where we left off."

AI (using MCP): 1. Calls resume_previous_session(ai_id="claude", count=1) 2. Loads the handoff report 3. Continues with full context

Result: No context loss, no repeated work, seamless continuation.

Troubleshooting

Server not connecting?

Check the path:

# Find your empirica installation
which empirica

# Use the full path in your MCP config
# Example: /Users/username/.venv/bin/empirica

Verify it runs:

# Test the MCP server directly
empirica mcp start

# You should see: "Empirica MCP Server started..."

Tools not showing?

Check your environment:

# Ensure you're in a project with empirica installed
pip list | grep empirica

# Check for errors
empirica --version

Check client logs: - Claude Desktop: ~/Library/Logs/Claude/mcp*.log - Cursor: .cursor/logs/ - Windsurf: .windsurf/logs/

Common Issues

Issue: "Command not found: empirica"
Solution: Use the full path to the Python executable in your MCP config.

Issue: "Tools appear but don't work"
Solution: Ensure you've bootstrapped a session first with bootstrap_session.

Issue: "Session not found"
Solution: Use resume_previous_session or create a new session with bootstrap_session.

Best Practices

For AI Assistants

  1. Always bootstrap first: Start every new conversation with bootstrap_session
  2. Be honest in assessments: Genuine self-awareness requires honest vector ratings
  3. Use handoffs: Create handoff reports for multi-session work
  4. Track goals: Use the goal system for complex tasks

For Users

  1. Let the AI assess: Don't skip PREFLIGHT, it's where self-awareness happens
  2. Review handoffs: Check handoff reports to understand what the AI learned
  3. Use session IDs: Reference specific sessions for continuity
  4. Monitor calibration: Check calibration reports to see if the AI's confidence matches reality
